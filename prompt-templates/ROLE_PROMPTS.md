# Role Prompts for Advanced Game Research

Use these meta-prompts inside the automatic pipeline. Replace variables from `CONTEXT.md`. All roles work on `{OUTPUT_FILE}` directly to build a singular, massive, cohesive research report.

## Section Executor Prompt (For EPIC-001 to EPIC-004)

Use this generic prompt for executing the analytical steps.

```markdown
Use the executor skill named in the current EPIC file.

Read:
- `prompt-templates/PLAN.md`
- `prompt-templates/CONTEXT.md`
- `[current EPIC file]`
- `{OUTPUT_FILE}` (if it already exists to maintain context continuity)

Task:
Research, analyze, and write ONLY the specific section owned by the current EPIC. You are acting as a senior specialist for this specific domain.

Output target:
Update `{OUTPUT_FILE}` directly. Append your section sequentially.

Required section pattern:
### [EPIC Name] BLUF
### Data & Evidence Analysis
### Framework Application
### Risks and Open Questions

Rules:
- Keep the scope strictly within the boundaries of the current EPIC.
- Cite or label every factual claim or metric. Use `[ESTIMATE]` for guessed metrics.
- Do not conclude the final recommendation yet; leave that for the Synthesis Editor.
- Hand off to `qa-evaluator` immediately upon completion.
```

## QA Evaluator Prompt

```markdown
Use the qa-evaluator skill.

Read:
- `prompt-templates/CONTEXT.md`
- `[current EPIC file]`
- `{OUTPUT_FILE}`
- `EVALUATION_CHECKLIST.md`

Task:
Evaluate only the newly added section generated by the previous agent. Compare it strictly against the EPIC's Acceptance Criteria (AC) and Definition of Done (DoD).

Output:
Return `[PASS - DELIVER]` or `[FAIL - BUG]` in the pipeline response.

Rules:
- If `[FAIL - BUG]`, you must explicitly cite which AC or DoD item was violated (e.g., "BUG: Hallucinated CPI metric without [ESTIMATE] tag").
- Do not evaluate formatting outside of what is requested in the EPIC.
```

## Final Synthesis Editor Prompt (For EPIC-005)

```markdown
Use the product-strategist skill.

Read:
- `prompt-templates/CONTEXT.md`
- `prompt-templates/epics/EPIC-005-lean-gdd-synthesis.md`
- `{OUTPUT_FILE}` (which now contains EPICs 001 through 004)

Task:
Act as the Executive Producer. Read the entirety of `{OUTPUT_FILE}`. Synthesize the findings into a finalized, actionable Lean Game Design Document (Lean GDD).

Output:
Update `{OUTPUT_FILE}` with the final sections.

Rules:
- Add the Executive Thesis (GREENLIGHT or KILL).
- Consolidate all previously listed Risks into a "Master Risk Ledger".
- Define the absolute Minimum Viable Product (MVP) features based on the findings.
- Do not introduce new unsourced claims that were not present in EPICs 001-004.
```
